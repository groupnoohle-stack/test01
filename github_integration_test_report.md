## 支持的主要功能

GitHub CLI 集成提供了丰富的功能，包括但不限于：

### 仓库操作
- 列出、查看、创建、克隆、删除仓库
- 管理仓库设置和权限
- Fork 和归档仓库

### Issue 和 PR 管理
- 创建、查看、编辑、关闭 issues
- 创建、审查、合并 pull requests
- 管理标签、里程碑和项目

### 代码协作
- 查看和创建 gists
- 管理代码审查
- 处理评论和反馈

### API 访问
- 通过 `gh api` 命令直接调用 GitHub REST API
- 支持 JSON 数据处理和过滤（使用 jq）

### 工作流和 Actions
- 查看和管理 GitHub Actions 工作流
- 触发和监控 CI/CD 流程

### 其他功能
- 管理组织和团队
- 处理通知
- 配置 webhooks
- 管理 releases 和 tags

## 优势特点

1. **预配置免登录**：GitHub CLI 已预先配置并认证，可以立即使用，无需手动设置
2. **完整的 API 访问**：支持所有 GitHub REST API 端点
3. **自动化友好**：适合脚本化和批量操作
4. **数据持久化**：克隆的仓库和本地更改在沙箱会话中保持
5. **安全性**：使用 token 认证，不暴露密码
